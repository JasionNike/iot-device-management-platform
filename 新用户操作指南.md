# 智能物联设备管理平台 — 新用户操作指南

> 如果你是第一次使用这个系统，跟着本文档 5 分钟就能上手。

---

## 一、这个系统是干什么的？

一个管理 IoT 智能设备的 Web 平台。支持：
- 📡 **设备接入**：注册和管理成千上万台物联网设备
- 🔄 **设备影子**：云端维护设备状态快照，断网也不丢数据
- ⚡ **OTA 升级**：远程批量升级设备固件，支持灰度发布
- 🚨 **告警中心**：设备异常自动检测告警，闭环处理

---

## 二、启动系统

### 方式一：一键启动（推荐）

```
双击 start.bat
```

脚本会自动启动 Redis、RabbitMQ、MySQL 检查、编译项目、启动后端、打开前端。

### 方式二：手动启动

```bash
# 1. 确保 MySQL、Redis、RabbitMQ 已运行

# 2. 首次运行：初始化数据库
mysql -u root -p123456 < sql/schema.sql
mysql -u root -p123456 < sql/data.sql

# 3. 启动后端
cd backend
mvn package -DskipTests
java -Dfile.encoding=UTF-8 -jar target/iot-platform-1.0.0-SNAPSHOT.jar

# 4. 打开前端
浏览器打开 frontend/index.html
```

### 端口说明

系统采用微服务架构，启动后各服务端口分配如下：

| 服务 | 端口 | 说明 |
|------|:----:|------|
| Gateway（网关统一入口） | 8080 | 前端只需访问此端口，自动路由到后端服务 |
| Backend（后端核心服务） | 8081 | 设备接入、影子、告警、OTA等业务 |
| Notification（通知服务） | 8082 | 告警通知发送，短信/邮件/推送 |
| Nacos控制台 | 8848/nacos | 服务注册与配置管理 |

### 启动后检查

浏览器访问以下地址确认服务正常：

1. **http://localhost:8080/** — Gateway欢迎页，列出所有可用服务链接
2. **http://localhost:8080/api/device/dashboard** — 设备统计大盘JSON数据

如果看到上述页面返回正常数据，说明启动成功：
```json
{"code":200,"data":{"totalDevices":203,"onlineDevices":201,...}}
```
> **提示**：系统内置设备模拟器（DeviceSimulator），启动后自动模拟200台MOCK设备持续上报遥测数据、触发告警事件、发送心跳。无需手动造数据，仪表盘和告警中心的数据会实时变化。

浏览器打开以下地址：
  平台欢迎页   : http://localhost:8080/
  设备统计大盘  : http://localhost:8080/api/device/dashboard
  Nacos控制台   : http://localhost:8848/nacos（需Java 17）
  通知服务健康  : http://localhost:8082/api/notification/health

## 三、前端界面介绍

访问 `http://localhost:8080/` 进入登录页（演示账号：admin/admin123）。登录后进入主界面，左侧导航栏包含 **7个功能模块**：

| 模块 | 功能说明 |
|------|---------|
| 工作台 | 设备统计卡片（总数/在线/离线/在线率）+ 设备类型饼图 + 在线状态柱状图 + 最近告警 |
| 设备管理 | 分页列表（按产品/状态筛选、关键字搜索）、注册设备、详情弹窗、影子查看、遥测历史图表 |
| 设备影子 | 输入设备SN查询影子JSON（reported/desired），支持JSON编辑下发期望状态 |
| 告警中心 | 告警列表（P0-P3筛选）、处理告警（确认/忽略/修复）、等级分布饼图、通知补发状态 |
| OTA升级 | 固件管理（上传/列表）、升级任务（创建/灰度推进）、升级统计（已升级/待升级） |
| 运行监控 | 遥测数据曲线图（按属性动态切换）、设备类型分布饼图、固件版本柱状图 |
| 产品管理 | 产品型号CRUD（创建/编辑/删除）、能力模型弹窗（属性/事件/指令各自独立增删改查） |

所有数据从后端API实时获取，页面顶部显示当前登录用户，支持退出登录。

## 四、核心功能操作流程

### 4.1 注册一台设备

**界面操作：** 点击"注册设备" → 填写设备名称、选择产品型号、输入SN、填写位置 → 确认

**API调用：**
```bash
curl -X POST http://localhost:8080/api/device/register \
  -H "Content-Type: application/json" \
  -d '{"deviceName":"温湿度传感器-01","productKey":"SENSOR-TH-001","deviceSn":"SN-001","location":"西安机房A"}'
```

**返回结果：** 系统自动生成 `deviceId`（全局唯一ID）和 `deviceSecret`（认证密钥），后续设备通信用。

### 4.2 查看设备影子

**什么是设备影子？**  
每个设备在云端都有一个 JSON 文档，存储设备最新上报的状态(reported) 和 平台期望的状态(desired)。

**界面操作：** 点击设备行的"影子"按钮

**API调用：**
```bash
curl http://localhost:8080/api/device/shadow/{deviceId}
```

**返回示例：**
```json
{
  "deviceId": "d5d81257...",
  "version": 2,
  "state": {
    "reported": {"temperature": 26.5, "humidity": 60.2, "battery": 85},
    "desired": {"target_temperature": 24.0}
  },
  "metadata": {
    "reported": {"temperature": {"timestamp": 1783066407013}, ...},
    "desired": {"target_temperature": {"timestamp": 1783066407083}}
  }
}
```

### 4.3 模拟设备上报数据

设备上报遥测数据后，影子自动更新：

```bash
curl -X POST http://localhost:8080/api/shadow/reported/{deviceId} \
  -H "Content-Type: application/json" \
  -d '{"temperature":28.5,"humidity":55.0,"battery":80}'
```

### 4.4 向设备下发指令

平台通过影子的 `desired` 字段向设备下发期望状态：

```bash
curl -X POST http://localhost:8080/api/shadow/desired/{deviceId} \
  -H "Content-Type: application/json" \
  -d '{"target_temperature":22.0,"power_status":"off"}'
```

设备下次上线或定时拉取影子时，会发现 `desired` 中有新的期望值，然后执行相应操作。

### 4.5 创建 OTA 升级任务

```bash
curl -X POST http://localhost:8080/api/ota/task \
  -H "Content-Type: application/json" \
  -d '{"taskName":"固件v2.0升级","firmwareId":"FW-V2","productKey":"SENSOR-TH-001","targetType":"ALL","grayPercent":10}'
```

### 4.6 查看告警

```bash
curl http://localhost:8080/api/alert/list?pageNum=1&pageSize=10
```

## 六、常见问题

### Q1: 前端页面显示"Network Error"

**原因：** 后端未启动或 CORS 跨域。

**解决：** 
1. 确认访问 http://localhost:8080/api/device/dashboard 有返回
2. 确认后端启动无报错

### Q2: 设备注册成功但列表显示 0

**原因：** 前端未刷新。

**解决：** 点击"查询"按钮刷新，或刷新浏览器页面。

### Q3: 影子数据不显示

**原因：** 设备还没有上报过数据。

**解决：** 先调用 `POST /api/shadow/reported/{deviceId}` 模拟设备上报，再查看影子。

### Q4: 启动脚本闪退

**原因：** MySQL 连接失败或 Redis/RabbitMQ 启动异常。

**解决：** 
1. 确认 MySQL 服务已启动
2. 确认 Redis 路径：`D:\soft\Redis-x64-5.0.9\`
3. 确认 RabbitMQ 路径：`D:\softPackage1\RabbitMQ Server\rabbitmq_server-3.12.14\`

### Q5: 接口返回中文乱码

**原因：** curl 在 Windows Git Bash 中发送中文 JSON 有编码问题。

**解决：** API测试时使用英文内容，或使用 Postman 等工具。

### Q6: 为什么访问 http://localhost:8080/ 看到的是欢迎页而不是404？

**回答：** Gateway添加了IndexController欢迎页，列出所有可用的服务链接。

### Q7: 访问接口返回401 Unauthorized？

**回答：** 大部分API需要Bearer Token认证。curl加 -H "Authorization: Bearer iot-platform-demo-token-2024"

### Q8: Nacos控制台打不开？

**回答：** Nacos 3.1.1需要Java 17，确认JDK17已安装。start.bat会自动用JDK17启动Nacos。
